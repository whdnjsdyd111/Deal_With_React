            2.5 ESLint와 Prettier 적용

        2.5.1 ESLint

ESLint는 문법 검사 도구고, Prettier는 코드 스타일 자동 정리 도구다.

ESLint는 코드를 작성할 때 실수하면 에러 혹은 경고 메시지를 VS Code 에디터에서 확인할 수 있게 해준다.
에디터 상단 메뉴에서  보기 > 문제  를 클릭하여 열 수 있다.


        2.5.2 Prettier

JSX를 작성할 때는 코드의 가독성을 위해 들여쓰기를 사용한다.
들여쓰기가 제대로 되어 있지 않은 코드는 읽기가 매우 힘들기 때문이다.

코드가 난잡할 경우  F1   를 눌러서 format이라고 입력한 다음 엔터를 눌러보자.
Prettier를 사용하여 자동 코드 정리를 하면 코드가 제대로 정렬되고, 세미콜론(;)이 빠진 곳에는 자동 추가가 될 것이다.

Prettier의 장점은 이런 스타일을 커스터마이징할 수 있는 것이다.
현재 열려 있는 프로젝트의 루트 디렉터리에서 .prettierrc 라는 파일을 생성한 후 다음 내용을 입력해보자.


{
    "singleQuote": true,
    "semi": true,
    "useTabs": false,
    "tabWidth": 2
}


이 파일은 들여쓰기를 할 때 대신 공백을 두칸 사용하도록 했다.
그리고 큰따옴표 대신 작은 따옴표를 쓰게 했고, 세미콜론은 언제나 붙이도록 설정했다.
Prettier에선 이 외에도 다양한 코드 스타일을 사전 설정할 수 있다.
자세한 내용은 Prettier Option 페이지를 참고하자.


	2.5.2.1 저장할 때 자동으로 코드 정리하기

코드 스타일을 정리할 필요가 있을 때마다  F1  을 누르고 Format을 입력하거나 단축키로 정리하는 것보다
편하게 정리할 수 있다.

먼저 VS Code 환경 설정을 연다.
설정은  파일 > 기본 설정 > 설정  메뉴를 클릭하여 열 수 있다.
상단 텍스트 박스에 format on save 를 검색하여 나오는 체크 박스를 체크해준다.
